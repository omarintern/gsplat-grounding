gsplat-grounding

Drop a character into a Gaussian-splat scene and have them walk on the visible floor — no authored collider, no preprocessing. The same .spz acts as both the visual environment and the collision source.

★ GitHub How it works API

Abstract

Gaussian-splatting captures look great but are notoriously hard to animate against — a splat is a cloud of fuzzy ellipsoids, not a manifold mesh. Today, putting a character into a splat scene means hand-authoring a collider in Blender, or relying on a separately shipped collision mesh.

gsplat-grounding derives a usable floor surface directly from the splat itself, in the browser, at load time. We sample where the splats are, figure out which ones make up the floor, and use that as the surface a character walks on. Each frame, whichever foot is lowest gets snapped to that surface — so the planted foot stays planted, the swing foot lifts naturally, and the character doesn't float or sink.

The whole library is one short JavaScript file. It works with any splat file the popular three.js splat renderer can read (.spz, .ply, .splat, .ksplat), and with any rigged character — Mixamo or otherwise.

How it works

Step 1 — Read where every splat is

A Gaussian splat scene is, mathematically, a cloud of fuzzy ellipsoids. Each ellipsoid has a center point. We read those center points (about 30,000 of them) when the splat first loads. They outline every visible surface in the scene — the floor, the walls, the furniture, the ceiling.

Step 2 — Build the heightmap

For every cell, sort all nearby splats by Z and take p15 (top of the lowest 15%). Sub-floor outliers fall below; furniture and ceilings sit above. Empty cells are filled from neighbours.

Step 3 — Each frame, snap the character to the floor

The character moves through the scene every frame — they walk, they wave, their hips bob, their feet leave the ground and come back. Whatever the animation does, we want one fact to stay true: whichever foot is the lowest right now should be touching the floor.

So each frame we:

  1. Look up the floor height at the character's current (x, y).
  2. Measure the lowest point on the character's body in their current pose.
  3. Lift or drop the whole character so that lowest point sits exactly on the floor.

The result: the planted foot stays put, the swing foot lifts naturally, and hips bob without the character ever floating above the ground or sinking through it.

three.js implementation notes

For a Mixamo-rigged SkinnedMesh animated by an AnimationMixer: update the mixer first so bones reflect this frame's pose, then take the precise bounding box — Box3.setFromObject(walker, true) — which transforms every skinned vertex into world space rather than reusing the rest-pose box. Bilinear-sample the heightmap at (walker.x, walker.y) for floorZ, then shift walker.position.z by floorZ − bbox.min.z.

Results

Tested on Marble Labs splats (indoor restaurants, halls, captures) and on mkkellogg's published demo scenes. Single-floor flat-ish indoor spaces are the strongest case. The library handles mild slopes in the captured floor by following them — the character traces the splat-derived terrain.

API

The library is one file. Five functions:

import {
  sampleSplatCenters,   // pull centroids out of an mkkellogg splatMesh
  buildSplatFloorHM,    // build the 2D floor heightmap once
  sampleSplatFloor,     // bilinear lookup at (x, y)
  groundCharacter,      // per-frame: snap walker to heightmap
  buildFloorCarpet,     // wireframe debug viz of the heightmap
} from './splat-grounding.js';

const samples = sampleSplatCenters(viewer.splatMesh, 30000);
const floorHM = buildSplatFloorHM(samples);

function tick() {
  mixer.update(dt);
  groundCharacter(walker, floorHM);
  renderer.render(scene, camera);
}

Strengths and limits

✓ works for
  • flat-ish indoor floors (rooms, galleries, restaurants)
  • outdoor terrain with smooth elevation
  • any rig three.js can bound (Mixamo, custom)
  • any splat format @mkkellogg/gaussian-splats-3d decodes
  • zero preprocessing, zero authored assets
✗ doesn't handle
  • multi-floor environments (stairs, balconies)
  • walking under tables / overhangs
  • walls / lateral collision (only floor)
  • very sparse splats (<8 samples per cell)

The single-floor limit is fundamental to z = f(x, y) — extending to stairs and balconies needs a multi-Z heightmap (store all dense z-bands per cell; walker picks the band closest to its current z). On the roadmap.

Built on

  • @mkkellogg/gaussian-splats-3d — does all the splat decoding, sorting, and rasterization. We only consume splatMesh.getSplatCenter().
  • three.js — renderer, Box3.setFromObject(obj, true) (precise skinned bbox), AnimationMixer.
  • Mixamo — the demo character rig and walk animation (the actual .glb isn't shipped; bring your own).
